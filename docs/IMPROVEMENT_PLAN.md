**当前问题**：基础HTML表格，无现代化设计，操作不便

## 二、数据格式扩展（向后兼容）

### 2.1 买卖关联关系设计（核心）

#### 2.1.1 关联逻辑
```
购买记录 (purchase)
    │
    ├── 赎回记录 1 (redeem) ──→ purchase_record_id = 购买记录ID
    ├── 赎回记录 2 (redeem) ──→ purchase_record_id = 购买记录ID
    └── 赎回记录 N (redeem) ──→ purchase_record_id = 购买记录ID

支持关系：
- 一对一：一次买入 → 一次全额赎回
- 一对多：一次买入 → 多次部分赎回
- 待赎回：一次买入 → 尚未赎回（持有中）
```

#### 2.1.2 状态计算规则
```python
def calculate_purchase_status(purchase, redeems):
    """计算购买记录的状态"""
    total_redeemed = sum(r['redeem_amount'] for r in redeems)
    remaining = purchase['amount'] - total_redeemed
    
    if remaining <= 0:
        return "completed"      # 已完结（全部赎回）
    elif total_redeemed > 0:
        return "partial"        # 部分赎回
    elif is_expired(purchase):
        return "expired"        # 已到期未赎回
    else:
        return "holding"        # 持有中
```

#### 2.1.3 汇总计算
```python
def calculate_summary(purchase, redeems):
    """计算投资汇总信息"""
    total_redeemed = sum(r['redeem_amount'] for r in redeems)
    total_profit = sum(r['actual_profit'] for r in redeems)
    remaining = purchase['amount'] - total_redeemed
    
    # 真实年化收益率（加权平均）
    if total_profit > 0 and total_redeemed > 0:
        weighted_days = sum(
            r['redeem_amount'] * days_between(purchase['purchase_date'], r['redeem_date'])
            for r in redeems
        ) / total_redeemed
        real_rate = (total_profit / total_redeemed) * (365 / weighted_days) * 100
    else:
        real_rate = 0
    
    return {
        "total_redeemed": total_redeemed,
        "remaining": remaining,
        "total_profit": total_profit,
        "real_rate": real_rate,
        "redeem_count": len(redeems)
    }
```
